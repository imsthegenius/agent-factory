## Overview

An agent provider defines how Narukami Shrine interacts with a specific AI coding tool. Each provider declares:

- An **env manifest** -- which environment variables it requires
- An **env check** -- validation that runs before the agent starts
- A **Dockerfile template** -- used during `narukami init` to scaffold the sandbox

## Built-in Providers

### Codex

The default agent provider. Uses Codex as the AI coding agent inside the sandbox, with `gpt-5.5` and low reasoning effort by default.

Default authentication:

| Variable         | Description                                                |
| ---------------- | ---------------------------------------------------------- |
| `~/.codex` mount | Reuses the host `codex login` ChatGPT subscription session |

### Claude Code

Uses Claude Code as an alternate AI coding agent inside the sandbox.

Required environment variables:

| Variable            | Description       |
| ------------------- | ----------------- |
| `ANTHROPIC_API_KEY` | Anthropic API key |

### OpenCode

Uses OpenCode as the AI coding agent. OpenCode output is passed through as raw stdout (no JSON parsing).

Required environment variables:

| Variable          | Description                    |
| ----------------- | ------------------------------ |
| Provider-specific | Depends on your OpenCode setup |

## Custom Providers

Agent providers are swappable. To add support for a different AI coding tool, create a new provider that implements the agent provider interface with its own env manifest, env check, and Dockerfile template.
