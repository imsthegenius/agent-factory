## What is Narukami Shrine?

Narukami Shrine is a TypeScript CLI tool for running Codex against Linear-tracked work inside isolated sandbox environments. It manages the full lifecycle of syncing code into a sandbox, invoking the agent, and syncing changes back.

## Key Concepts

- **Sandbox** -- An isolated environment (Docker container or local directory) where an agent executes code
- **Host** -- Your machine where Narukami Shrine runs and the real git repo lives
- **Agent** -- The AI coding tool invoked inside the sandbox (e.g. Codex)
- **Iteration** -- A single invocation of the agent, producing at most one commit

## Quick Start

### 1. Install

```bash
npm install -g @yae-tools/narukami-shrine
```

### 2. Initialize

Run `narukami init` in your repository to scaffold the config directory:

```bash
narukami init
```

This creates a `.narukami/` directory with a Dockerfile, prompt file, and configuration.

### 3. Run

```bash
narukami
```

Narukami Shrine will build the Docker image, sync your code into the sandbox, and start Codex.
